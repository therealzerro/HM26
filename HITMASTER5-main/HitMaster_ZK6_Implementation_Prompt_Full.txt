3) INGESTION — HISTORY (ADMIN ONLY)
Goal: Persist validated annual history for Box and each Pair class; compute percentiles and update coverage.
Input formats:
• BOX: { Combo|ComboSet, TimesDrawn, Expected?, LastSeen(YYYY‑MM‑DD), DrawsSince(int≥0) }
• PAIR: { Pair("00"…"99"), TimesDrawn, LastSeen(YYYY‑MM‑DD), DrawsSince(int≥0) } with class_id 2..11
Validation & normalization:
• DrawsSince ≥ 0; LastSeen parseable; strip numeric commas.
• Keys: pairs zero‑padded; box digits sorted ascending inside braces.
Persistence steps:
1) Write import record with counts + status=“validating”; fail fast on schema errors (line refs).
2) Upsert rows into datasets_box / datasets_pair for (class_id, scope, horizon_label, key_*).
3) Compute p99 cap for ds_raw within the set; winsorize; compute percentile map DS→[0,1]; upsert into percentile_maps.
4) Update horizon_blends coverage for the class (append new horizon label). Mark import status=“committed” and log to audit_logs.
UI/UX:
• History lane shows accepted/rejected/fixed counts; p99 value; first/last dates; keys normalized examples.

4) INGESTION — DAILY INPUT (ADMIN ONLY)
Goal: Apply same‑day deltas from the official feed; these are Box‑shaped rows for combos that hit today.
Process:
1) Validate as Box shape; resolve to comboset_sorted.
2) For each combo ABC in the payload: set DS=0 for {A,B,C} across active horizons within scope; increment DS+1 for all other sets in scope.
3) Emit pair events for ordered pairs (AB, BC, AC) and unordered any‑pos pairs ({A,B},{A,C},{B,C}) for all 10 pair classes (based on class semantics). Update their DS similarly.
4) Update percentile maps incrementally or mark for nightly refresh (OK).

5) INGESTION — LEDGER (ADMIN ONLY; AUTHORITATIVE)
Goal: Persist actual results with session & jurisdiction to power DS updates and exclusions.
Process:
1) Upsert each row by composite key (jurisdiction|date_et|session|result_digits). Reject duplicates.
2) Derive comboset_sorted and ordered/unordered pairs.
3) Run the same DS updates as Daily Input (reset hitters to 0; others +1) within the scope affected.
4) Mark same‑day exclusion flags for straight combos and their box sets for that date+scope.
5) Log to audit_logs with counts and duration.

6) NORMALIZATION & P99 WINSORIZATION (CLASS×SCOPE×HORIZON)
• For each (class_id, scope, horizon_label):
  – Compute p99 cap for ds_raw; clip values.
  – Convert to percentile rank in [0,1] (ties allowed; mid‑rank preferred).
• Persist p99 and map into percentile_maps; expose read RPC (no private internals).
• Refresh cadence: on import commit; nightly batch safe for rebalancing.
Guardrail: Winsorize BEFORE percentile mapping to prevent dominance by long tails.

7) HORIZON BLENDING (SHORT‑HEAVIER)
• For each class, assemble available horizons (H01Y…H10Y) and compute a geometric short‑heavier weight vector (example pattern acceptable as long as it sums to 1 and decays with age).
• Blend normalized values across horizons to produce a single class score component per key (BOX set or PAIR key).
• Cache blend weights and horizons_present in horizon_blends. Recompute only when a new horizon arrives.

8) ZK6 COMPONENTS (ALL NORMALIZED)
For each straight combo ABC:
• BOX*  — percentile DS for comboset {A,B,C} after horizon blend for the Box class.
• PBURST* — take the best permutation (x,y,z) of ABC maximizing mean(Front*(xy), Back*(yz), Split*(xz)), where * indicates the class‑blended, percentile‑normalized value for the corresponding pair classes.
• CO* — any‑position unordered pair pressure = mean(percentile of {A,B}, {A,C}, {B,C} across appropriate classes).
Multiplicity prior (optional):
• Singles +0.00, Doubles −0.02, Triples −0.04 to smooth extreme cluster dominance.

9) RAILS (CONSTRAINTS & EXCLUSIONS)
• Same‑day exclusion: remove any straight or box set that hit today in the active scope (from ledger).
• Quotas for K6 slate: Singles ≤ 4, Doubles ≤ 2 (Triples off or gated).
• Pair‑repetition cap: ≤ 2 items sharing the same TopPair across K6.
• Mid↔Eve recency guard: apply scope‑local rules to avoid echoing within the same calendar day.
• If mandatory H01Y is missing for any class: class contribution = 0 and flag “partial data” in the status ribbon.

10) INDICATOR & TEMPERATURE
Indicator(ABC) = w1*BOX* + w2*PBURST* + w3*CO* + multiplicityPrior
• Default weights: 0.40 / 0.40 / 0.20 (Admin can later tune in Scoring Lab)
• Conservative alt: 0.85 / 0.10 / 0.05
Temperature 0–100:
• Map Indicator through distribution cut‑points (p10→0, p90→100), with clipping outside range.

11) SLATE ASSEMBLY & SNAPSHOTS
Process:
1) Score all eligible combos in scope.
2) Apply rails (exclusions, quotas, pair cap) and sort by Indicator desc with deterministic tie‑breakers (Box DS at longest horizon, then lexicographic).
3) Produce K6 + backfill list; compute Temperature for display.
4) Persist slate_snapshot with:
   – updated_at_et (server in ET), horizons_present (per class), weights_json, top_k_straights_json, top_k_boxes_json (optional), and a hash over content for determinism.
5) Keep a rolling window of N snapshots per scope. Expose a read‑only view for clients.
